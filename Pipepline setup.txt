pipeline {
    agent any
    
    stages {
        stage("clone code") {
            steps {
                git url: "https://github.com/junaidtyagi9555/Docker-Projects.git", branch: "main"
            }
        }
        stage("build docker img") {
            steps {
                sh "docker build -t node-todo-app ."
            }
        }       
        stage("deploy container") {
            steps {
                sh """
                docker stop node-app-container ||  true
                docker rm node-app-container || true
                docker run -d -p 8000:8000 --name node-app-container node-todo-app
                """
            }
        }
}
}



If interviewer asks:

What is Jenkins Pipeline?

Answer:

Jenkins Pipeline is a CI/CD workflow defined as code using a Jenkinsfile, enabling automated build, test, and deployment stages.

If asked:

Why use Pipeline instead of freestyle?

Answer:

Pipeline provides version control, scalability, and better CI/CD automation compared to freestyle jobs.
